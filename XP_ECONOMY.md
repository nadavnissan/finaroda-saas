# FINARODA SaaS — XP Economy (v1.0, נעול 11/07/2026)

> סוגר את החוב הפתוח (Onboarding Spec §8 / ALIGNMENT D3). כפוף ל: UX §5 (Status Tiers, מתוקן כאן) · UX §9.4 (איסור gamification של תדירות) · CLAUDE.md §8.3.
> עיקרון-על: **XP מודד משמעת ולמידה. לעולם לא תוצאות, לעולם לא תדירות.**

## 1. מקורות XP — רשימה סגורה (כל מקור אחר אסור בלי עדכון מסמך זה)

| מקור | XP | תדירות/הגנה |
|---|---|---|
| השלמת אונבורדינג (S0–S11) | 300 | חד-פעמי, תסריטאי (‏50+100+50+100) |
| סריקה ראשונה של היום | +50 | פעם ביום (D3). סריקות נוספות = 0, ללא cooldown וללא הודעת "פספסת" |
| שיעור אקדמיה הושלם | +100 | פעם אחת לשיעור, מלאי סופי |
| צפייה בתוצאת יומן שנחשפה | +25 | פעם אחת לתרחיש; קצוב מבנית (תרחישים נפתרים בקצב השוק) |
| מענק תמיכה (אדמין בלבד) | משתנה | **לא נצבר ע"י המשתמש** — כלי תמיכה של אדמין בלבד, מתועד ב-`admin_events` עם מזהה-אדמין+סיבה (audit). idempotent per (user, `admin_grant`, ref). לא מוצג ב-"HOW XP IS EARNED" כי המשתמש אינו יכול להרוויחו. הוסף ב-B7 (Package B phase 2, פריים B7b) — פיצוי תקלה וכד'. |
| **אסור לצמיתות:** רווח/תוצאת תרחיש, מספר סריקות, רצפים, הזמנת חברים-תמורת-XP | 0 | עיקרון חוקתי — הפרה = הפרת RED LINE תרבותית |

## 2. Streak — הוחלט: לא קיים ב-v1
+50 היומי מתגמל את ההרגל לינארית: יום שפוספס אינו מוחק דבר. streak מוסיף צוק ו-loss-aversion — בדיוק מה ש-UX §9.4 אוסר. שורת ה-streak המכוכבת ב-B5 **נמחקת**. בחינה מחדש רק אם דאטת ה-100 תראה כשל בהתבססות ההרגל — ואז גרסה שבועית רכה, מגובת מספרים, בעדכון מסמך זה.

## 3. סולם דרגות (v1)

| # | דרגה | סף XP | פותח (תוכן-בונוס, אורתוגונלי לשערי פלאן) |
|---|---|---|---|
| 01 | Strategy Apprentice | 0 | — |
| 02 | Risk Manager | 1,000 | ספריית "Spike Autopsies" — ניתוחי אפיזודות אמת |

| 04 | Master Strategist | 8,000 | תג + מקום ראשון בתור לפיצ'רים בבטא |
קצב ייחוס: משתמש יומי מעורב ≈ ‏2,500 XP/חודש → דרגה 02 בשבועיים, 04 ב~5 חודשים.

## 4. תיקון UX §5 (סתירה שנמצאה)
דרגת "מדייק" הותנתה ב"היסטוריית what-if חיובית" — תגמול על תוצאות, סותר את סעיף 1. **מתוקן:** דרגות = XP בלבד. איכות ה-what-if נשארת סטטיסטיקת דשבורד, לעולם לא קריטריון דרגה.

## 5. תצוגה ותחולה
- XP ודרגה **פרטיים** ב-v1: אין leaderboard, אין השוואה בין משתמשים. call-sign + תג = הזהות.
- זהה בכל הפלאנים כולל Free (נעול).
- תקרת בטיחות טכנית: אירוע XP נרשם צד-שרת בלבד, idempotent per (user, source, day/lesson/scenario).

## 6. השלכות מיידיות
1. B5 (Design): מחיקת שורת ה-streak; ספי הדרגות 1,000/3,000/8,000 מחליפים את הכוכביות; שורת "+25 journal reveal viewed" נוספת ל-"HOW XP IS EARNED".
2. UX §5: עדכון קריטריון "מדייק" (סעיף 4 כאן).
3. מימוש: טבלת xp_events (user, source, ref, amount, ts) + unique constraint.
